Parte 1

Lo que hace el código dado es iterar sobre los frames de un video, elegir uno de ellos como fondo, y posteriormente obtener la diferencia binaria entre el fondo y el fotograma actual, aplicar una erosión morfológica a la imagen para eliminar el ruido y seguidamente detectar todas las regiones de la imagen, estas regiones se ordenan por su área, y si la región mayor es más grande que cierto límite escogido se asigna como centroide, es decir, se considera que se ha detectado movimiento en ese punto.

El código se encuentra en la carpeta BS.

Respecto al código, la primera solución que hemos utilizado es la más simple, utilizar una imagen de fondo elegida a mano, y simplemente convertir a escala de grises el fondo y todas las imágenes en cada iteración y simplemente calcular las regiones de diferencia como:

foreground = abs(input_image-image_template)>th;

En las imágenes podemos observar a partir de un análisis visual que el objeto en movimiento aparece en 6 de las 10 imágenes, utilizando como template o fondo la primera imagen (“imagen0000.png”) y jugando un poco con el threshold para que no detecte pequeños cambios, con thresholds de alrededor de 0.2 podemos obtener un muy buen resultado que ubique correctamente la posición del pájaro en 5 de las imágenes, para obtener el resultado óptimo para este caso, ya que la primera aparición del pájaro por el borde izquierdo es imposible de detectar con estos ajustes hemos cambiado también otros parámetros, aunque esta solución está muy sobre ajustada para este caso(overfitting), hemos cambiado también el factor de erosión para que no borre la primera aparición del pájaro como si fuera ruido, y también el área mínima de la región para contabilizar regiones más pequeñas., la configuración final queda tal que:

th=0.05;
bwmorph(foreground,'erode',1);

M1

El algoritmo GC, lo primero que hace, es, cargar la imagen, una vez cargada la imagen, utiliza la función Superpixels(), que agrupa los píxeles en regiones (super pixeles) en función de valores similares de escalera de grises/RGB.

Una vez tenemos los super pixeles, haremos (mediante 2 puntos), diferentes rectángulos, en los que distinguiremos el foreground del background (verde para el foreground y rojo para el background). Una vez hechos los rectángulos, haremos una mask binaria para el foreground y el background, y posteriormente, haremos servir la función lazysnapping(), donde introduciremos las masks, los super pixeles y la imagen real, y con esta  función obtenemos una segmentación entre el foreground y background.

Operamos el algoritmo, en primer lugar, tal y como viene dado en la práctica, en el que podemos seleccionar, 1 rectángulo para el foreground y dos para el background, podemos ver el resultado de la segmentación para las 2 imágenes,  podemos ver que al no poder delimitar con precisión (al utilizar únicamente 3 rectángulos), el algoritmo, nos da una segmentación poco precisa.

Modificamos un poco el algoritmo para poder hacer más rectángulos y poder delimitar mejor el foreground y el background, y podemos ver como claramente, el resultado mejora, y el algoritmo segmenta con más precisión las imágenes.

Volvemos a modificar el algoritmo, reduciendo el nombre de super píxeles, de 500 a 200, y como podemos ver, mejora todavía un poco más la precisión en la segmentación, y se debe a que en estos casos, agrupar por regiones más grandes, ayuda a la segmentación.

Para reducir la interacción del usuario, podríamos utilizar formas geométricas distintas al rectángulo, como podría ser la unión de distintos puntos para delimitar mejor el foreground y el background, y así con una única forma geométrica, delimitar el foreground del background, ya que podríamos, considerar background todo lo que no es foreground, y con esta única forma evitaremos el uso repetitivo de rectángulos, reduciriamos la interacción del usuario y mejoraríamos la segmentación.

Parte 3:

La segmentación de video es el proceso de dividir un video en partes semánticamente significativas. El clustering de subespacios es un método que agrupa puntos de datos en múltiples subespacios de manera que obtenemos puntos que están muy cerca si pertenecen a cada subespacio o  puntos que se encuentran muy separados si no pertenecen a los subespacios.

M1:

La función main.m es el punto de entrada para el algoritmo de segmentación de escenas de vídeo. Llama a otras funciones para poder generar los datos que se utilizarán como input, generar una matriz de afinidad Z llamando a osc_relaxed.m para obtener los datos con filtros de diferente orden (1,2,4).  Más tarde llama a nCutW.m para generar los clústers y con el vector de clúster generados,  los compara con el Ground truth (resultado verdadero) para observar la cantidad de error existente.

La función osc_relaxed.m resuelve un problema de optimización para recuperar la matriz de afinidad para la segmentación. La matriz de afinidad representa las similitudes entre los puntos de datos y se utiliza para agrupar los datos en diferentes segmentos. Aquí previamente filtraremos los datos según el orden escogido (1,2,4).

La función ncutW.m interpreta la matriz de afinidad en diferentes grupos. Utiliza el criterio de corte normalizado para asignar cada punto de datos a un grupo, lo que resulta en la segmentación de las escenas de vídeo.

La segmentación semántica se refiere a la tarea de clasificar cada píxel de una imagen en una de varias clases predefinidas. La segmentación de instancias, por otro lado, se refiere a la tarea de no solo clasificar cada píxel en una de varias clases predefinidas, sino también diferenciar entre instancias individuales de la misma clase.
Observando el siguiente frame se ha decidido hacer una clasificación entre diferentes objetos en la escena: boxeadores (azul y rojo), referee, background generando polígonos para cada uno de estos y guardandolo en un archivo .JSON este nos permitirá asociar los objetos en diferentes frames a tiempo real.
